# Contributing to NIST Compliance Guide

Thank you for your interest in contributing to this NIST compliance reference guide! This document provides guidelines for contributions.

## Ways to Contribute

### Adding New NIST Publications

1. **Verify the source** - All resources must be from the official `nist.gov` domain
2. **Include required links:**
   - CSRC publication page (e.g., `https://csrc.nist.gov/pubs/sp/...`)
   - PDF download link (e.g., `https://nvlpubs.nist.gov/...`)
3. **Add context** - Brief description of the publication's purpose and relevance

### Updating Existing Content

- Check that existing links are still valid
- Update version numbers when new revisions are published
- Add new control mappings or implementation examples

### Improving Documentation

- Fix typos or clarify confusing sections
- Add examples for control implementations
- Improve the structure or organization

## Submission Guidelines

### For Small Changes

1. Fork the repository
2. Make your changes
3. Submit a pull request with a clear description

### For Larger Contributions

1. Open an issue first to discuss the proposed changes
2. Reference the issue in your pull request

## Content Standards

### Link Format

Use this format for NIST resources:

```markdown
| Resource | URL |
|----------|-----|
| **Main Publication** | https://csrc.nist.gov/pubs/... |
| **PDF Document** | https://nvlpubs.nist.gov/... |
```

### Control Mapping Format

```markdown
| Control | Title | Example Implementation |
|---------|-------|------------------------|
| XX-## | Control Title | Brief implementation description |
```

### Checklist Items

Use unchecked boxes for reusable checklists:

```markdown
- [ ] Checklist item description
```

## Quality Guidelines

1. **Accuracy** - Verify all information against official NIST publications
2. **Currency** - Note publication dates and check for updates
3. **Clarity** - Write for readers who may be new to NIST standards
4. **Objectivity** - Present information without vendor bias

## What NOT to Include

- Proprietary or vendor-specific implementations
- Links to non-NIST sources (except official standards bodies)
- Legal or compliance advice
- Unofficial interpretations of standards

## Code of Conduct

- Be respectful and constructive in discussions
- Focus on improving the guide for all users
- Credit original sources appropriately

## Questions?

Open an issue for questions about contributing or the content of this guide.
